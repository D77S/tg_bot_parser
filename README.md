Парсер сайтов с уведомлением в ТГ-бот, если обнаружится изменение.

На данный момент настроено, что парсятся три сайта. Из них для одного парсинг временно отключен, пока не требуется.

Для каждого сайта в настройках задаются данные для парсинга.


При инициализации:
- загружаются переменные среды
- запускается логгинг
- проверяется наличие первых двух переменных - токен бота и айди чата хозяина
- если их нет - аварийный выход
- в бот хозяина пишется, что начата инициализация серверной части парсера
- проверяется проверяется наличие всех остальных переменных среды
- если их нет - аварийный выход с сообщением в бот хозяина
- заполняется двумерный список с настройками сайтов для парсинга, из переменных среды
    - адреса сайта
    - текстовое название для сайта
    - периодичность парсинга
    - флаг запрета на парсинг в нерабочее время и в нерабочие дни (часовой пояс Москва)
    - некоторые текстовые константы
    - функция распарсинга по тегам (зашито жестко в хардкод программы)
- попытка инициализации пустого списка результатов парсинга
- если неуспешно, то сообщение в бот хозяина и аварийный выход
- пустой список результатов заполняется первичным результатом парсинга всех сайтов, невзирая на флаг запрета выходных дней. В элемент списка вносится словарь, в нем под ключом moment текущее время, под ключом data словарь результатов парсинга (определяется функцией распарсинга)
- запускается бесконечный цикл
    - перебираются по кругу все элементы списка из сайтов
        - определяется текущий момент времени
        - определяется, рабочее ли время в данный момент
        - если нерабочее и для данного сайта стоит флаг запрета - пауза 1 сек и переход к следующему сайту
        - если с момента прошлого парсинга данного сайта прошло больше времени, чем период его парсинга, то:
            - выгружаются из хранилища результаты прошлого парсинга данного сайта
            - производятся три попытки получить ответ сайта, с экспоненциальным нарастанием времени ожидания
            - если ответа так и нет - переход к следующему сайту
            - попытка распарсинга полученного ответа сайта
            - если неуспешна - переход к следующему сайту
            - результаты распарсинга сравниваются с предыдущими, из хранилища
            - если различаются - то
                - в бот хозяину посылаются оба результата, и новый и старый
                - в хранилище обновляется результат распарсинга на текущий
            - в хранилище обновляется время на текущее
        - пауза 1 сек.
